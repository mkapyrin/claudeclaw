# Правила для AI-аудита и оптимизации Яндекс.Директ

Эти правила обязательны. При конфликте с общими знаниями — приоритет у этих правил.

## РСЯ (Рекламная Сеть Яндекса)

### Таргетинг
- РСЯ таргетирует по ИНТЕРЕСАМ и ПОВЕДЕНИЮ пользователя, НЕ по поисковым запросам
- Автотаргетинг (---autotargeting) — нормальная и даже предпочтительная настройка для РСЯ
- Яндекс ML в РСЯ часто работает лучше ручного таргетинга

### Что НЕ работает в РСЯ
- Минус-слова НЕ ВЛИЯЮТ на показы — они фильтруют по содержанию страниц, не по запросам. Режут охват без пользы
- Title2 НЕ ПОКАЗЫВАЕТСЯ — в РСЯ выводится только Title1 + Text + Картинка

### Нормы метрик РСЯ (НЕ сравнивать с поиском!)
- CTR норма: 0.3-1.0%
- CTR > 2% = подозрение на фрод/скликивание (если bounce > 80% и нет конверсий)
- CTR > 2% при bounce < 60% и есть конверсии = хорошие креативы, НЕ фрод
- CTR < 0.05% = креативы не работают
- Bounce 50-70% = НОРМАЛЬНО для РСЯ, не реагировать
- Bounce > 80% = мусорные площадки, рекомендовать чистку
- CPC норма: 5-15 руб

### Оптимизация РСЯ
- Качество картинок и креативов — главный рычаг
- Тексты объявлений, аудитории, площадки
- Отключение плохих площадок (через интерфейс Директа)
- При росте CPA — снизить бюджет на 20-30%, НЕ приостанавливать (потеряется обучение)

### ЗАПРЕТЫ для РСЯ (NEVER)
- Рекомендовать минус-слова
- Рекомендовать Title2
- Оценивать CTR по нормам поиска
- keyword_cleanup (в РСЯ нет поисковых запросов)

## Поиск

### Таргетинг
- Таргетинг по ПОИСКОВЫМ ЗАПРОСАМ пользователя
- Группировка ключей по микротемам: 5-10 ключей на группу
- Типы соответствия (широкое, фразовое, точное) влияют на охват и релевантность
- Автотаргетинг допустим как дополнение к ручным ключам, но не единственный источник

### Минус-слова (ОБЯЗАТЕЛЬНЫ)
- Минимум 50-100 минус-слов на кампанию
- Для мягких ниш: бесплатно, скачать, реферат, курсовая, диплом, торрент, вакансия, работа, зарплата
- Минус-слова влияют ТОЛЬКО на поисковые кампании

### Title2 (ВАЖЕН)
- Показывается в расширенном формате объявления
- Использовать для доп. преимуществ, УТП

### Нормы метрик Поиска
- CTR норма: 2-5%
- CTR > 5% = хорошо
- CTR < 2% = проблема
- CTR < 0.5% при >100 показов = критично
- Bounce > 60% при CTR > 2% = проблема с ПОСАДОЧНОЙ, не с ключами

### Диагностика поисковых кампаний (по КОМБИНАЦИИ метрик)
- CTR < 1% + bounce > 60% = нерелевантная семантика, нужны минус-слова
- CTR > 2% + bounce < 40% = хорошая кампания, можно масштабировать (+20% бюджет)
- 0 конверсий + clicks > 50 = проблема с посадочной или семантикой
- 0 конверсий + clicks < 50 = МАЛО ДАННЫХ, не трогать
- Позиция > 4 = сузить семантику (бюджет и ставки НЕ влияют на позицию при автостратегии)

## Стратегия оптимизации

### Автостратегии (95% кампаний)
- Приостановка СБРАСЫВАЕТ обучение — suspend только как крайняя мера (фрод, CPA > 3x целевого)
- Для снижения расхода: уменьшить бюджет на 20-30%, НЕ приостанавливать
- Увеличивать бюджет НЕ более 20% за раз (резкое изменение сбивает автостратегию)
- Масштабирование: сначала увеличить бюджет работающей кампании, потом создавать новую

### Перераспределение бюджетов (главный рычаг)
- Смотри CPA по КАЖДОЙ кампании, НЕ средний CPA
- Бюджет перетекает от дорогих к дешёвым: CPA кампании A = 300 руб, B = 60 руб — снижай A, увеличивай B
- Общий бюджет должен оставаться примерно постоянным
- Автопилот ДОЛЖЕН уметь снижать бюджеты, не только увеличивать
- Не увеличивать бюджет если CPA кампании > медианного CPA среди кампаний с конверсиями
- Приоритет: масштабировать кампании с НИЗКИМ CPA

### Метрики — что считать проблемой
- CPA > 3x от лучшей кампании = рекомендовать снижение бюджета
- Нет конверсий за 7+ дней при > 50 кликах = серьёзная проблема
- Bounce rate > 70% на поиске = проблема с релевантностью
- Не трогать кампании с < 3 дней данных или < 30 кликов

### Безопасность
- Максимум 5 действий за цикл оптимизации
- Принцип: если не уверен — ai_analysis, не suspend
- Не повторять действия из списка последних действий
- Не рекомендовать ai_analysis для PAUSED кампаний с < 10 кликов

## API подводные камни

### Бюджеты
- Авто-стратегии используют WeeklySpendLimit, а НЕ DailyBudget
- WeeklySpendLimit: TextCampaign.BiddingStrategy.Search/Network
- Для получения: getCampaigns с TextCampaignFieldNames: ['BiddingStrategy']
- Деньги в API: micros (x 1_000_000), в UI — рубли

### Reports API
- IncludeVAT: 'NO' — чтобы цифры совпадали с UI Директа (рекламодатель видит без НДС)
- Conversions = уникальные конверсии (1 на визит), UI показывает ВСЕ достижения целей
- Для точных конверсий: Goals: [goalId] + AttributionModels: ['AUTO'] (массив, мн.число!)
- AttributionModel (ед.число) — НЕ СУЩЕСТВУЕТ, ошибка 8000
- AttributionModels работает только с CAMPAIGN_PERFORMANCE_REPORT и CRITERIA_PERFORMANCE_REPORT
- CUSTOM_REPORT НЕ поддерживает AttributionModels
- returnMoneyInMicros: 'false' — деньги в рублях

### Баланс аккаунта
- GetBalance (v4) — УСТАРЕЛ, ошибка 509
- Правильно: AccountManagement через live v4 API
- URL: https://api.direct.yandex.ru/live/v4/json/ (НЕ .com, НЕ /json/v4/)
- Action: "Get" НЕ требует finance_token
- Amount приходит как СТРОКА, нужен Number(Amount)

### Определение типа кампании
- По имени надёжнее чем по DB type: /РСЯ|rsy|yan/i = YAN, /Поиск|search/i = SEARCH
- DB type может быть неправильным для AI-созданных кампаний

### Конверсии: расхождение UI vs API
- UI Директа "303 конверсии" = сумма ВСЕХ достижений целей (1 визит может дать 3+ целей)
- Reports API Conversions = уникальные визиты с конверсией
- Для совпадения с UI: Goals: [goalId1, goalId2, ...] + AttributionModels: ['AUTO']

## Антипаттерны (NEVER делать в рекомендациях)
- Рекомендовать минус-слова для РСЯ (бесполезно)
- Рекомендовать Title2 для РСЯ (не показывается)
- Оценивать CTR РСЯ по нормам поиска (разные миры)
- Увеличивать бюджет дорогих кампаний вместо перераспределения
- Показывать расход с НДС (рекламодатель видит без НДС)
- Рекомендовать изменение ставок при автостратегиях
- Приостанавливать кампании вместо снижения бюджета

## Реальные бенчмарки кампаний ШОО (Январь-Март 2026)

**Январь 2026 (5 кампаний РСЯ):**
- Всего: ~200K показов, ~900 кликов, ~10K₽ расход
- Стратегия: в основном клик-кампании

**Февраль 2026 (10 кампаний РСЯ):**
- Всего: ~500K показов, ~2000 кликов, ~20K₽ расход, ~150 конверсий
- Конверсионные кампании: CPA ~49₽
- Кликовые кампании: CPA ~231₽ (в 4.7 раза дороже!)

**Март 2026 (неделя 1):**
- CPC снизился с 69₽ до 21₽ за неделю (обучение алгоритма)
- 100% конверсий с мобильных устройств

## Ключевые инсайты аудитории

- Ядро ЦА: женщины 25-45
- Лучший сегмент: женщины 35-44 (CR 21.4%)
- Второй сегмент: женщины 25-34 (CR 13.5%)
- Мужчины: 0 конверсий
- 96% конверсий с мобильных устройств

## Критические выводы по воронке

- Конверсионные кампании в 4.7 раза эффективнее кликовых по CPA
- 89% бюджета тратилось на кликовые кампании (ошибка)
- ROI при прямых продажах: +840% до +1318%
- ROI при текущей воронке (бот): от -4% до +59%

## Правила оптимизации бюджета (специфика ШОО)

- Перераспределить бюджет: 80%+ на конверсионные кампании
- Менять бюджет не более чем на 20% за раз (не убить обучение)
- Не ставить автостратегию на паузу (убивает обучение)
- Мобильный трафик — приоритет
